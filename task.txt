rest api - json

GET /users - list all the users in HTML format - DONE
GET /api/users - list all the users in json - DONE


GET /api/users/1 - get the user with id 1 - DONE
GET /api/users/2 - get the user with id 2 - DONE
// dynamic parameters
GET /api/users/:id - this ":id" is dynamic value and it can be anything, because we cannot make different routes for different userId's


POST /api/users - create new user

PATCH /api/users/1 - edit the user with id 1

DELETE /api/users/1 - delete the user with id 1
